File Buttons
New clears the workspace after a confirmation prompt — your blocks will be lost if not saved first. Save XML saves the current blocks as a .xml file you can reload later with Load. Save .py exports the generated MicroPython code as a .py file. Load opens a previously saved .xml file and restores the blocks into the workspace.
▶ Load Example ▾
Load Example
Click the Load Example button to open a drop-down menu of built-in example programs, organized by category. Selecting an example clears the workspace and loads the example blocks — a confirmation prompt will appear first so you don't lose your work by accident. Examples include: Hello Lights (pixels + tones), Motor Run & Stop, Wheel Rotations, Servo Sweep, Pixel Light Show (animations), Play a Tune, Move Forward & Back, and Square Pattern (turns).
🔗 Connect
Connect
Opens the connection dialog. Choose Bluetooth (BLE) to connect wirelessly — your browser will show a list of nearby controllers; select your SEEKER device. Choose USB / Serial to connect via a data cable — select the correct COM port from the browser prompt. Once connected, the button turns green and shows the connection type. Click it again to disconnect.
● Slot
1 — Red ▼
⬆ Upload to Slot
Program Slot Selector & Upload
The controller can store up to 12 programs at once, each in a numbered slot. Use the slot dropdown to choose which slot (1–12) to upload your program to. The colored dot updates to match the slot's color. Once connected, slots that already contain a program are shown in green with a ● in the dropdown — empty slots are plain. Click Upload to Slot to send the current blocks to the selected slot. The upload compiles your blocks to MicroPython, writes it to the controller as prog_N.py, and automatically restarts the launcher. Each upload takes a few seconds over Bluetooth.
🤖 Launcher
How the Program Launcher Works
When the controller powers on it runs a built-in launcher. The launcher lets you browse and run any of the 12 stored programs without connecting to the IDE. Use the buttons on the controller to navigate:

Button A — step backward through slots
Button B — step forward through slots
Start — run the selected slot's program

The four NeoPixels show the current slot's color as you scroll. An occupied slot glows at full brightness; an empty slot is the same color but dim. When the controller first boots, a slot map is printed to the console showing all 12 slots and whether each one is occupied. When a program finishes (reaches Ag End), the launcher restarts automatically and is ready to run again.
■ Stop / Reset
Stop / Reset
Immediately stops any running program, turns off motors, sound, and pixels, and restarts the launcher. Use this any time the robot is behaving unexpectedly or you need to return to the slot selector without waiting for the program to finish. Stop / Reset works over both Bluetooth and USB.
▶ Console
✕ Clear
Console
The Console tab shows everything printed by the controller — connection messages, the slot map at startup, output from Print blocks in your programs, and any errors. Use Print blocks in your program to display sensor readings or variable values here while your program runs. The ✕ Clear button in the top-right corner of the console wipes the output. You can also type raw MicroPython commands into the input bar at the bottom of the console and press Send (or Enter) to run them directly on the controller.
⚙ MicroPython
MicroPython Tab
Shows the MicroPython code that your blocks generate, updated live as you build. You don't need to read or edit this — it's there so you can see exactly what gets uploaded to the controller. It's also a great way to start learning how the code maps to the blocks.

Set Movement Speed
Sets the default speed used by all Move and Start blocks. Range is 0–100%. The default value is 50%. Set this once before your first movement command.
🤖 Set Acceleration / Deceleration rate
15
Set Acceleration / Deceleration
Sets a global ramp rate applied automatically by all movement blocks when they start and stop. A value of 0 means instant start/stop (default — no ramp). Higher values ramp faster: 10 is a gentle ramp, 20 is moderate, 30 is quick. A rate of 15 takes about 400 ms to reach 60% speed. Set once at program start alongside Set Movement Speed. Ramping reduces wheel slip, improves straight-line accuracy, and makes turns more repeatable — especially at higher speeds.
